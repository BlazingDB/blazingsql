Detailed Description

This interface represents a computation unit in the execution graph. Each kernel has basically and input and output ports and the expression asocciated to the computation unit. Each class that implements this interface should define how the computation is executed. See run() method.

◆ run()

virtual kstatus ral::cache::kernel::run ( )
pure virtual

Executes the batch processing. Loads the data from their input port, and after processing it, the results are stored in their output port.

Returns
kstatus 'stop' to halt processing, or 'proceed' to continue processing.

Implemented in ral::batch::OutputKernel, ral::batch::Print, ral::batch::Filter, ral::batch::Projection, ral::batch::BindableTableScan, ral::batch::TableScan, ral::batch::LimitKernel, ral::batch::MergeStreamKernel, ral::batch::JoinPartitionKernel, ral::batch::PartitionKernel, ral::batch::MergeAggregateKernel, ral::batch::SortAndSampleKernel, ral::batch::PartwiseJoin, ral::batch::DistributeAggregateKernel, ral::batch::ComputeWindowKernel, ral::batch::PartitionSingleNodeKernel, ral::batch::ComputeAggregateKernel, and ral::batch::UnionKernel.
